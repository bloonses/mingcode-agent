SECTION 01 / 05

不止 ReAct。
四框架状态机。

单一 ReAct 只能"走一步看一步"。MINGCODE 用状态机调度 4 种认知框架， 复杂任务先规划再执行，执行后反思，失败自动重规划，不确定时主动向用户提问。

▸ FRAMEWORK 01 · PLANNING

Plan-and-Execute

复杂任务先拆解为子任务列表，再逐个执行。Planner 内嵌思维树（ToT）， 生成 N 个候选方案 → LLM 评分 → 选最优 → 解析为可执行步骤。

core/planner.py · _generate_candidates / _evaluate / _select_best
▸ FRAMEWORK 02 · REFLECTION

Self-Reflection

每个任务执行后 LLM 反思，识别假成功（result 含 Error / Traceback 时判定 fail）。 失败按 retries 计数升级降级链，避免在死胡同里反复挣扎。

core/reflector.py · 假成功检测 + L1/L2/L3 分级降级
▸ FRAMEWORK 03 · THINKING

ToT 思维树

规划阶段生成多个候选方案，LLM 横向评估对比，选出最高分方案再展开为任务列表。 候选分隔符 === Candidate N === 解析，兜底单任务保证可降级。

core/planner.py · _split_candidates + _evaluate
▸ FRAMEWORK 04 · SELF-ASK

Self-Ask

Executor 每轮 ReAct 后调 SelfAsker 检测不确定性。uncertain 时调 ask_user 工具 向用户提问，回答塞回 memory 作为 clarification。失败兜底，不阻断主循环。

core/self_asker.py · ~100 token LLM 判断 + ask_user

L1 · LOCAL RETRY 局部重试 单任务失败时回到 EXECUTING 重试，受 max_task_retries 上限约束（默认 2 次）。
L2 · REPLAN 整体重规划 超出 L1 后回到 PLANNING，把失败 feedback 喂给 Planner 重新规划，受 max_replans 约束（默认 3 次）。
L3 · ESCALATE 报错给用户 超出 L2 后进入 DONE，把失败原因上报用户。CognitiveController 异常时 fallback 走 ReAct，零感知升级。
SECTION 02 / 05

一个终端，
全部能力。

从 Shell 到桌面控制，从微信到 QQ 官方 Bot，从短期记忆到自我进化的长期记忆。 每一项能力都可独立工作，组合起来即是完整的编码代理。

CAP · 01
多供应商 LLM
兼容任何 OpenAI 格式 API。Ollama / OpenAI / DeepSeek / Qwen / Zhipu / Moonshot 一键切换， base_url 改一下就能用。reasoning_effort 参数支持推理模型思考深度（low/medium/high）。
ollama deepseek qwen
CAP · 02
15+ 内置工具
Shell、文件读写编辑、Python 执行、网络搜索、Git、HTTP、数学、时间、待办、子智能体、 ask_user、plan_tot、桌面控制。ToolRegistry 统一注册调度，OpenAI Function Calling 标准。
shell files git search
CAP · 03
Computer Use
截屏 + vision LLM 分析屏幕元素（带坐标）+ 鼠标键盘自动化。4 步工作流： screenshot → click/type → screenshot 验证 → 循环。三路独立降级，截图用完即删。
vision pyautogui RPA
CAP · 04
微信 / QQ 接入
微信 ClawBot（iLink 协议长轮询）、QQ OneBot 11（NapCat/Lagrange WebSocket）、 QQ 官方 Bot（AccessToken + 心跳 + 扫码 AES-GCM 绑定）。远程也能用同一个 Agent。
wechat qq onebot qq official
CAP · 05
子智能体 Subagent
主 Agent 在 ReAct 循环中可自主调用 task 工具派生子智能体。独立上下文记忆， 有限递归（depth=2，主 → sub → sub），经验共享长期记忆池，daemon thread 超时兜底。
recursive isolated shared ltm
CAP · 06
自我进化记忆
四类长期记忆：preference / project / success / lesson。从错误中自动学习，跨会话保留。 关键词检索 + 多维评分（标签重叠 ×2 + 子串匹配 + 使用频率 + 最近使用）。threading.Lock 并发安全。
preference success lesson
CAP · 07
赛博极简 UI
霓虹青绿色调（#00ff88），锐利直角边框，打字机流式输出，代码语法高亮。 Rich + Pygments 后端，box.SQUARE 直角边框，ANSI truecolor 半块字符渲染截图缩略图。
rich neon typewriter
CAP · 08
一键安装分发
PyInstaller 打包成单文件 exe，Inno Setup 制作图形化安装程序。 自动配置 PATH、开始菜单快捷方式，普通用户不需要管理员权限，控制面板可正常卸载。
pyinstaller inno setup

// ARCHITECTURE

组合优于继承，
状态机驱动.

NeonAgent 是主入口，cognitive_controller property 首次访问才构造 CognitiveController， 异常时 fallback 走 ReAct，保证向后兼容。SubAgent 独立类组合复用主 LLM 和 LongTermMemory， 不继承 NeonAgent，递归深度由 depth 参数控制。

TodoList 在 NeonAgent 创建一次，AI 工具（TodoTool）和用户命令（/todo）注入同一实例， 这是"AI 与用户协同"的核心模式。所有认知框架组件都遵循 stub → 真实实现的可降级升级路径。
